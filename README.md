# 🤖 Multi-Agent System

> **Multi-Agent System** — это передовая веб-платформа для проектирования, настройки и выполнения сложных сценариев взаимодействия интеллектуальных ИИ-агентов с ролевыми моделями. Проект представляет собой полноценное Full-Stack приложение, спроектированное по архитектуре **Monorepo** (монорепозиторий). Интерфейс включает интерактивный визуальный граф связей (XYFlow/React Flow), изолированную песочницу для выполнения кода и продвинутую RAG-систему работы с документами.

---

## 🚀 Стек технологий

*   **Backend Core:** Python, FastAPI, Uvicorn.
*   **Database & RAG:** SQLite + SQLAlchemy, ChromaDB с кастомным TF-IDF / Cosine Similarity фолбеком для отказоустойчивости.
*   **Frontend Core:** Next.js (App Router), React, TypeScript, Zustand (минималистичное хранилище состояния).
*   **Styling & UI:** Tailwind CSS + Vanilla CSS, XYFlow/React, Framer Motion.
*   **AI Integrations:** OpenRouter API.
*   **Auth & Security:** JWT (JSON Web Tokens) с использованием библиотеки `PyJWT` для безопасной аутентификации.
*   **Containerization & Sandbox:** Docker & Docker Compose (контейнеризация сервисов приложения и запуск изолированной песочницы выполнения кода на базе образа `python:3.11-slim`).

## ✨ Ключевые возможности

### 1. 📊 Визуальный граф воркфлоу
*   Проектирование связей и сценариев взаимодействия между агентами с использованием интерактивного холста на базе XYFlow.
*   Стилизованные анимированные ребра связей (`AnimatedEdge.tsx`) для наглядного отображения активных потоков обмена сообщениями и зависимостей в реальном времени.

### 2. 🧠 Мультиагентный оркестратор
*   Ядро оркестрации (`orchestrator.py`), управляющее памятью сессии, системными промптами агентов, ролевыми моделями и распределением задач.
*   Асинхронный стриминг ответов LLM через OpenRouter API для бесшовного вывода текста в реальном времени.

### 3. 🛡️ Защищенная песочница кода
*   Выделенный модуль `sandbox.py` для безопасного выполнения сгенерированного агентами Python-кода непосредственно в серверном окружении.
*   Динамический захват стандартного вывода (stdout/stderr) с последующей трансляцией результатов и ошибок во встроенную панель терминала.

### 4. 📂 Интеллектуальный RAG-модуль
*   Поддержка чтения и парсинга документов форматов PDF (через `PyPDF`), Excel (`pandas`), CSV и обычных текстовых файлов.
*   Векторизация и семантический поиск по фрагментам документов с использованием векторной базы данных ChromaDB.
*   **Умный фолбек:** автоматическое переключение на встроенный легковесный поисковый движок на базе TF-IDF и косинусного сходства при отсутствии запущенной БД ChromaDB, что гарантирует работу RAG в любых условиях.

### 5. 🔑 Безопасность и хранение данных
*   Надежная регистрация и авторизация пользователей по стандарту JWT (JSON Web Tokens).
*   Сохранение конфигурации агентов, структуры графа, файлов и логов в асинхронную базу данных SQLite.

## 🔒 Безопасность и архитектурные решения

1.  **Единый коммерческий стандарт комментариев:** В основном проекте весь исходный код снабжен комментариями на английском языке. В текущей демонстрационной версии (Showcase) комментарии к коду оставлены на русском языке исключительно для ознакомительных целей, чтобы упростить рекрутерам и техническим специалистам детальный разбор архитектурных решений.
2.  **Изоляция CORS & Сетевая безопасность:** На бэкенде настроена middleware-защита для обмена данными только с доверенными адресами (в целях локальной разработки разрешены все хосты `*`, для продакшена конфигурируется конкретный домен фронтенда).
3.  **Хранение секретов:** Конфиденциальные данные (API-ключи, JWT секреты) хранятся исключительно в файлах `.env` и добавлены в `.gitignore`, что предотвращает их случайное попадание в публичный репозиторий GitHub.
